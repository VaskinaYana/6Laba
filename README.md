# Задание 1

### Текст задачи

Разработать класс с двумя логическими полями. Создать конструктор копирования. Разработать
метод, вычисляющий импликацию полей. Перегрузить метод ToString() для формирования
строки из полей класса. Реализовать дочерний класс (его содержание предложить
самостоятельно и описать в решении: какой содержательный смысл имеют поля; реализовать
конструкторы; предложить и реализовать 2-3 метода). Протестировать все конструкторы и
другие методы базового и дочернего классов.

### Алгоритм решения

1. Запуск программы 
2. Выводится приглашение "Введите first (true/false): ", пользователь вводит строку, которая преобразуется в тип bool через bool.Parse() и сохраняется в переменную a.
3. Выводится приглашение "Введите second (true/false): ", пользователь вводит строку, которая преобразуется в bool и сохраняется в переменную b.
4. Ввызывается конструктор LogicalPair(a, b), который сохраняет переданные значения в приватные поля first и second, создается объект obj.
5. Вызывается метод ToString() для obj, который возвращает строку "First = X, Second = Y"
6. Вызывается метод Implication() для obj, вычисляет логическое выражение !first || second
7. Вызывается конструктор копирования LogicalPair(obj), который копирует значения полей first и second из переданного объекта, создается объект copy.
8. Вызывается метод ToString() для copy и результат выводится на экран.
9. Выводится приглашение "Введите имя: ", пользователь вводит строку, которая сохраняется в переменную name.
10. Вызывается конструктор LogicalExpression(a, b, name), который через base(a, b) вызывает конструктор родительского класса для инициализации полей first и second, затем сохраняет переданное имя в поле name, создается объект expr.
11. Вызывается переопределенный метод ToString() для expr, который возвращает строку "Name = X, First = Y, Second = Z", результат выводится на экран.
12. Вызывается метод IsEqual() для expr, который сравнивает значения first и second через GetFirst() == GetSecond(), результат выводится на экран.
13. Вызывается метод Invert() для expr, который меняет значения полей на противоположные через SetValues(!GetFirst(), !GetSecond()).
14. Вызывается метод ToString() для expr и результат выводится на экран.
15. Вызывается конструктор копирования LogicalExpression(expr), который через base(other) копирует поля родительского класса (first и second), затем копирует поле name, создается объект exprCopy.
16. Вызывается метод ToString() для exprCopy и результат выводится на экран.

# Задание 2 - 3

### Текст задачи

Необходимо реализовать определение класса (поля, свойства, конструкторы, перегрузка
метода ToString() для вывода полей, метод: Вычитание копеек (uint) из объекта типа Money(учесть, что денежная величина не
может быть меньше 0). Результат должен быть типа Money ). Протестировать все методы, включая конструкторы. В задании 3 добавить к реализованному в задании 2 классу указанные в варианте перегруженные операции. Протестировать все методы.


### Алгоритм решения

1. Программа запускается и выводит приглашение для ввода первой денежной суммы.
2. Вызывается метод ReadUInt для ввода рублей с проверкой, что введено положительное целое число. 
3. Вызывается метод ReadByte для ввода копеек с проверкой, что число от 0 до 99.
4. Создается объект m1 через конструктор с параметрами, который сохраняет рубли и копейки (с проверкой, что копейки меньше 100).
5. Аналогично вводятся рубли и копейки для второй суммы и создается объект m2.
6. Выводятся исходные данные через метод ToString(), который возвращает строку вида X руб. Y коп.
7. Тестируется унарный инкремент ++: создается копия m1, к ней применяется ++, копейки увеличиваются на 1, при переполнении рубли увеличиваются на 1, а копейки обнуляются.
8. Тестируется унарный декремент --: создается копия m2, к ней применяется --, копейки уменьшаются на 1.
9. Проверяется работа -- на нулевой сумме - сумма не уходит в минус и остается 0 руб. 0 коп.
10. Тестируется явное приведение к uint через (uint)m1 - возвращаются только рубли, копейки отбрасываются.
11. Тестируется неявное приведение к bool - для m1 возвращается true (сумма не равна нулю), для нулевой суммы возвращается false.
12. Тестируется бинарная операция Money - uint: вводится количество копеек для вычитания, вызывается метод SubtractKopeks, который переводит всё в копейки, вычитает, и возвращает новый объект с результатом.
13. Тестируется бинарная операция uint - Money: вводится количество копеек, создается временный объект из этих копеек, из него вычитается сумма m2.
14. Тестируется бинарная операция Money - Money: обе суммы переводятся в копейки, выполняется вычитание, если вторая сумма больше или равна первой - возвращается 0.
15. Выполняются дополнительные тесты: вычитание большей суммы из меньшей (возвращается 0), многократный инкремент (0.98 → 0.99 → 1.00 → 1.01), многократный декремент (1.00 → 0.99 → 0.98 → 0.97).
