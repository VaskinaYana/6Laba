# Васькина Яна ИТС-2 Лабораторная №6

# Задание 1

### Текст задачи

Разработать класс с двумя логическими полями. Создать конструктор копирования. Разработать
метод, вычисляющий импликацию полей. Перегрузить метод ToString() для формирования
строки из полей класса. Реализовать дочерний класс (его содержание предложить
самостоятельно и описать в решении: какой содержательный смысл имеют поля; реализовать
конструкторы; предложить и реализовать 2-3 метода). Протестировать все конструкторы и
другие методы базового и дочернего классов.

### Алгоритм решения

1. Запуск программы.
2. Пользователь вводит строку, которая сохраняется в переменную name.
3. Пользователь вводит строку, которая преобразуется в тип bool через bool.Parse() и сохраняется в переменную isActive.
4. Пользователь вводит строку, которая преобразуется в bool и сохраняется в переменную isVerified.
5. Вызывается конструктор UserStatus, который сохраняет переданные значения в приватные поля _isActive и _isVerified, создаётся объект status.
6. Вызывается конструктор копирования UserStatus(status), который копирует значения полей _isActive и _isVerified из переданного объекта, создаётся объект statusCopy.
7. Вызывается метод ToString()
8. Вызывается конструктор User, который через base вызывает конструктор родительского класса UserStatus для инициализации полей _isActive и _isVerified, затем сохраняет переданное имя в поле _name, создаётся объект user.
9. Вызывается переопределённый метод ToString() для user, который через base.ToString() получает строку от родительского класса, добавляет имя и результат метода IsFullyAuthorized(), возвращает строку
10. Вызывается метод Implication() для user, который вычисляет логическое выражение !_isActive || _isVerified, результат выводится на экран.
11. Вызывается метод GetIsActive() для user, который возвращает значение поля _isActive, и проверяется условие. Если пользователь активен, то выводится сообщение "Блокировка" и вызывается метод Block(), устанавливает поле _isActive в false, оставляя _isVerified без изменений. Если пользователь не активен, то выводится сообщение "Разблокировка" и вызывается метод Unblock(), устанавливает поле _isActive в true, оставляя _isVerified без изменений.
12. Вызывается метод ToString() для user, результат выводится на экран, отображая изменённое состояние объекта.
13. Вызывается конструктор копирования User, который через base(other) вызывает конструктор копирования родительского класса для копирования полей _isActive и _isVerified, затем копирует поле _name из переданного объекта, создаётся объект userCopy.
14. Вызывается метод ToString() для userCopy, результат выводится на экран.

# Задание 2 - 3

### Текст задачи

Необходимо реализовать определение класса (поля, свойства, конструкторы, перегрузка метода ToString() для вывода полей, метод: Вычитание копеек (uint) из объекта типа Money(учесть, что денежная величина не может быть меньше 0). Результат должен быть типа Money ). Протестировать все методы, включая конструкторы. В задании 3 добавить к реализованному в задании 2 классу указанные в варианте перегруженные операции. Протестировать все методы.  


### Алгоритм решения

1. Программа запускается и выводит приглашение для ввода первой денежной суммы.
2. Вызывается метод ReadUInt для ввода рублей с проверкой, что введено положительное целое число. 
3. Вызывается метод ReadByte для ввода копеек с проверкой, что число от 0 до 99.
4. Создается объект m1 через конструктор с параметрами, который сохраняет рубли и копейки (с проверкой, что копейки меньше 100).
5. Аналогично вводятся рубли и копейки для второй суммы и создается объект m2.
6. Выводятся исходные данные через метод ToString(), который возвращает строку вида X руб. Y коп.
7. Унарный инкремент: создается копия m1, к ней применяется ++, копейки увеличиваются на 1, при переполнении рубли увеличиваются на 1, а копейки обнуляются.
8. Унарный декремент: создается копия m2, к ней применяется --, копейки уменьшаются на 1.
9. Проверяется работа -- на нулевой сумме - сумма не уходит в минус и остается 0 руб. 0 коп.
10. Тестируется явное приведение к uint через (uint)m1 - возвращаются только рубли, копейки отбрасываются.
11. Тестируется неявное приведение к bool - для m1 возвращается true (сумма не равна нулю), для нулевой суммы возвращается false.
12. Тестируется бинарная операция Money - uint: вводится количество копеек для вычитания, вызывается метод SubtractKopeks, который переводит всё в копейки, вычитает, и возвращает новый объект с результатом.
13. Тестируется бинарная операция uint - Money: вводится количество копеек, создается временный объект из этих копеек, из него вычитается сумма m2.
14. Тестируется бинарная операция Money - Money: обе суммы переводятся в копейки, выполняется вычитание, если вторая сумма больше или равна первой - возвращается 0.
